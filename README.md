# Frontend(React + Vite)
Фронтенд находится в папке `frontend/` и сделан как React-приложение на Vite. Навигация внутри приложения работает через hash-маршруты.

## Реализованные экраны

- `#` — главное меню работы с участниками.
- `#create` — форма ручной регистрации нового участника.
- `#database` — просмотр списка участников с поиском по ФИО и раскрывающимися карточками записей.
- `#edit/{student_id}` — форма редактирования записи участника.
- `#import` — форма загрузки участников из Excel-файла.

## Главное меню

На главной странице размещены переходы:

- `Добавить участника вручную` — открывает форму регистрации.
- `Загрузить участников из файла` — открывает форму импорта.
- `Просмотреть список участников` — открывает страницу базы участников.

## Регистрация участника

Форма регистрации находится в `frontend/src/pages/ParticipantRegistrationPage.jsx`.

Что сделано:

- данные разделены на вкладки: `Личная информация`, `Контактная информация`, `Документы`, `Банковские реквизиты`;
- вкладки закреплены в верхней части формы и адаптивно складываются при уменьшении ширины окна;
- внизу формы есть кнопки перехода `Назад` / `Далее` и кнопка `Создать запись`;
- добавлены маски и проверки для телефона, дат, паспорта, СНИЛС, ИНН и банковских реквизитов;
- при создании отправляется студент через `POST /students`;
- если заполнены банковские реквизиты, дополнительно создается запись через `POST /students/{student_id}/bank-details`.

## Просмотр списка участников

Страница списка находится в `frontend/src/pages/ParticipantsDatabasePage.jsx`.

Что сделано:

- загрузка списка участников через `GET /students`;
- поиск по ФИО;
- раскрывающаяся карточка участника с учебными, контактными, документными, служебными данными и банковскими реквизитами;
- банковские реквизиты загружаются при раскрытии записи через `GET /students/{student_id}/bank-details`;
- внутри раскрытой карточки добавлена кнопка `Редактировать запись`, которая ведет на `#edit/{student_id}`.

## Редактирование участника

Форма редактирования находится в `frontend/src/pages/ParticipantEditPage.jsx` и повторяет структуру формы регистрации.

Что сделано:

- загрузка данных участника через `GET /students/{student_id}`;
- загрузка банковских реквизитов через `GET /students/{student_id}/bank-details`;
- редактирование тех же вкладок и полей, что в форме регистрации;
- сохранение основных данных через `PATCH /students/{student_id}`;
- сохранение существующих банковских реквизитов через `PATCH /bank-details/{bank_details_id}`;
- если реквизитов еще нет, но пользователь заполнил банковский блок, создается новая запись через `POST /students/{student_id}/bank-details`.

## Импорт участников из файла

Страница импорта находится в `frontend/src/pages/ParticipantImportPage.jsx`.

Что сделано:

- создан экран по макету для импорта данных волонтеров;
- добавлен выбор Excel-файла (`.xlsx`, `.xls`);
- добавлены кнопки `Загрузить`, `Проверить документ`, `Вернуться назад`;
- добавлена нижняя темно-синяя область под будущий вывод логов.

Вывод логов и реальная обработка файла пока не реализованы.

## API-клиент

Запросы к backend собраны в `frontend/src/lib/api.js`.

Сейчас используются функции:

- `createStudent`;
- `getStudent`;
- `updateStudent`;
- `listStudents`;
- `createBankDetails`;
- `listBankDetails`;
- `updateBankDetails`.

Базовый URL API берется из `VITE_API_BASE_URL`, по умолчанию используется `/api/v1`.
После запуска приложения через docker доступ к странице можно получить по адресу http://localhost:3000/

# Backend (FastAPI + PostgreSQL)

REST API для формы: студенты и банковские реквизиты. Структура: `backend/` — приложение, `database/` — SQL-схема, `frontend/` — заглушка под будущий фронт.

## Запуск через Docker

1. Установите и запустите [Docker Desktop](https://www.docker.com/products/docker-desktop/) (Windows) или Docker Engine с Compose v2.

2. В терминале перейдите в корень проекта:

   ```powershell
   cd C:\папка проекта
   ```

3. Поднимите PostgreSQL и API (при изменениях в коде пересоберёт образ API):

   ```powershell
   docker compose up --build -d
   ```

4. Проверка:

   - API: [http://127.0.0.1:8000/docs](http://127.0.0.1:8000/docs) (Swagger)
   - Проверка без БД: [http://127.0.0.1:8000/health](http://127.0.0.1:8000/health)

5. Остановка контейнеров (данные БД в volume сохраняются):

   ```powershell
   docker compose down
   ```

6. Полный сброс данных БД (удалит том, при следующем запуске снова выполнится `database/schema.sql`):

   ```powershell
   docker compose down -v
   ```

### Переменные окружения

Скопируйте `.env.example` в `.env` и при необходимости измените логин/пароль БД, порты (`API_PORT`, `POSTGRES_PORT`) и `DATABASE_URL` для сервиса `api`.

### Только база данных

Если API запускаете локально (`uvicorn`), достаточно БД:

```powershell
docker compose up -d db
```

В `.env` для приложения укажите `DATABASE_URL` с `localhost` и тем же портом, что проброшен у сервиса `db` (по умолчанию `5432`).

### Если сборка образа падает по сети

Повторите `docker compose build api` или `docker compose up --build -d`. Ошибки TLS/таймаута при скачивании образов с Docker Hub обычно проходят после повторной попытки или при более стабильном интернете.

## Локальный запуск API без Docker

Из корня репозитория (нужен запущенный PostgreSQL и совпадающий `DATABASE_URL`):

```powershell
pip install -r requirements.txt
uvicorn main:app --reload
```

Альтернатива из папки `backend`: `uvicorn app.main:app --reload`.
